## 需求演化与功能扩展

项目最初目标是完成题目要求中的基础 CRDT 协同编辑与隐私视图机制。随着开发推进，系统从“基础树形协同编辑原型”逐步演化为“支持多角色、多部门、离线同步和前端演示能力的隐私保护协同编辑系统”。

主要演化过程如下：

| 演化方向 | 具体扩展 |
| --- | --- |
| CRDT 核心能力 | 从基础 JSON 树节点操作扩展到删除优先、级联删除、删除影响分析、冲突整理和并发视图同步验证。 |
| 隐私保护机制 | 从代码中的基础权限判断扩展为配置化 RBAC + ABAC 策略，支持节点级可见性过滤、字段级脱敏、多部门隔离和不可见节点路径处理。 |
| 多用户与账户体系 | 从测试用户切换扩展到密码登录、用户注册、会话管理、管理员用户管理和单点登录约束。 |
| 网络同步机制 | 从在线 WebSocket 同步扩展到离线操作队列、批量重放、重复操作去重、heartbeat 状态维护和重连后自动同步。 |
| 前端与演示能力 | 从简单 Web 树编辑页面扩展到操作日志、Markdown 编辑与预览、Undo/Redo、在线人数、节点编辑状态、用户资料菜单、设置弹窗和头像上传。 |
| 工程化与验证 | 从基础单元测试扩展到 CRDT 操作、视图变换、权限策略、服务端接口、离线 UI、冲突处理、并发同步、Markdown 和 Undo/Redo 等多层测试，并补充 Docker 部署配置。 |

### 最终演示角色

最终演示建议围绕多角色、多部门和隐私视图差异展开，重点展示“同一份完整 CRDT 文档在不同用户处生成不同视图，并且合法编辑能够同步回完整文档”的核心能力。

| 演示角色 | 展示重点 |
| --- | --- |
| `admin` 管理员 | 展示完整文档视图、用户管理、较高权限编辑能力和操作日志。 |
| `manager` 部门负责人 | 展示部门内节点可见、部分敏感字段受限、可执行部门范围内编辑操作。 |
| `member` 普通成员 | 展示只能访问本部门或本人相关节点，验证隐私视图过滤和权限感知编辑。 |
| `guest` 访客 | 展示仅能查看公开内容，验证低权限用户无法访问隐私节点和敏感字段。 |

演示流程可以采用“管理员查看完整文档 -> 不同角色登录对比视图差异 -> 普通用户进行合法编辑 -> WebSocket 同步到其他用户视图 -> 断网编辑后重连同步 -> 查看操作日志和在线状态”的顺序，集中体现 CRDT 协同、隐私保护和工程化扩展能力。
